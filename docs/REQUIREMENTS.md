# 要求定義書

## プロジェクト概要

Amazon風のECサイトを構築する。
ユーザーが商品の閲覧・購入ができる機能と、管理者が商品や在庫を管理できる機能を実装する。

## 技術構成

| 項目 | 技術 |
|------|------|
| フロントエンド | React（TypeScript） |
| ユーザー向けAPI | Ruby on Rails |
| 管理者向けAPI | Laravel（PHP） |
| データベース | MySQL 8.0 |
| セッション管理 | Redis |
| コンテナ | Docker / Docker Compose |
| CSS | Material-UI |
| テスト（Rails） | Minitest |
| テスト（Laravel） | PHPUnit |
| テスト（React） | Jest |
| E2Eテスト | 各フレームワークのテスト機能で実施 |

## ユーザー向け機能

### ユーザー管理
- 会員登録（メールアドレス、パスワード、氏名、電話番号）
- ログイン / ログアウト
- パスワードリセット
- プロフィール編集

### 商品関連
- 商品一覧表示（ページネーション付き）
- 商品詳細表示
- カテゴリー別表示
- 商品検索
- 在庫切れ商品の表示

### カート
- カートに商品を追加
- 数量変更
- 商品削除
- カート内容の表示

### 注文
- 配送先住所の選択
- 決済方法の選択（実際の決済は行わない）
- 注文確定
- 注文履歴の確認
- 同時購入時の在庫チェック

### 住所管理
- 住所の登録・編集・削除
- デフォルト住所の設定

## 管理者向け機能

### 認証
- 管理者ログイン
- 管理者アカウント作成

### 商品管理
- 商品の登録・編集・削除
- 画像アップロード
- 商品販売停止機能

### 在庫管理
- 在庫数の変更
- 在庫切れ商品の管理
- 在庫変更ログ

### 注文管理
- 注文一覧・詳細表示
- 注文ステータス更新

### カテゴリー管理
- カテゴリーの登録・編集・削除

### スケジュール機能
- 毎朝9時に在庫一覧をCSV出力する

## セキュリティ方針

- JWT認証によりAPI通信を保護する
- セッションはRedisに保存する（ファイルへの保存は要件上不可のため）
- パスワードはbcryptでハッシュ化を行う
- CORS設定により、許可されたオリジンのみアクセスを許可する
- 入力値はサーバー側で適切にバリデーションを行う

## テスト方針

Small / Medium / Big の3段階に分けてテストを実装する。

- **Small（単体テスト）**: モデルのバリデーションやビジネスロジックの検証を行う
- **Medium（結合テスト）**: APIのエンドポイントや認証フローの検証を行う
- **Big（E2Eテスト）**: APIレベルで一連の業務フローをテストする（ユーザー登録→カート→注文の流れなど）

データの設定のみの処理など、明らかに自明なもの以外はテストカバレッジを確保する。

## 設計方針

- コントローラーにロジックを集中させず、モデルやサービス層に適切に分離する
- RESTfulなAPI設計を採用する
- Makefileに頻出コマンドをまとめ、開発効率を高める
