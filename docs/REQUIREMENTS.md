# 要求定義書

## プロジェクト概要

Amazon風のECサイトを作成する。
ユーザーが商品を閲覧・購入できる機能と、管理者が商品や在庫を管理する機能を実装する。

## 技術構成

| 項目 | 技術 |
|------|------|
| フロントエンド | React（TypeScript） |
| ユーザー向けAPI | Ruby on Rails |
| 管理者向けAPI | Laravel（PHP） |
| データベース | MySQL 8.0 |
| セッション管理 | Redis |
| コンテナ | Docker / Docker Compose |
| CSS | Material-UI |
| テスト（Rails） | Minitest |
| テスト（Laravel） | PHPUnit |
| テスト（React） | Jest |
| E2Eテスト | Playwright |

## ユーザー向け機能

### ユーザー管理
- 会員登録（メールアドレス、パスワード、氏名、電話番号）
- ログイン / ログアウト
- パスワードリセット
- プロフィール編集

### 商品関連
- 商品一覧表示（ページネーション付き）
- 商品詳細表示
- カテゴリー別表示
- 商品検索
- 在庫切れ商品の表示

### カート
- カートに商品を追加
- 数量変更
- 商品削除
- カート内容の表示

### 注文
- 配送先住所の選択
- 決済方法の選択（実際の決済は行わない）
- 注文確定
- 注文履歴の確認
- 同時購入時の在庫チェック

### 住所管理
- 住所の登録・編集・削除
- デフォルト住所の設定

## 管理者向け機能

### 認証
- 管理者ログイン
- 管理者アカウント作成

### 商品管理
- 商品の登録・編集・削除
- 画像アップロード
- 商品販売停止機能

### 在庫管理
- 在庫数の変更
- 在庫切れ商品の管理
- 在庫変更ログ

### 注文管理
- 注文一覧・詳細表示
- 注文ステータス更新

### カテゴリー管理
- カテゴリーの登録・編集・削除

### スケジュール機能
- 毎朝9時に在庫一覧をCSV出力

## セキュリティ方針

- JWT認証でAPI通信を保護
- セッションはRedisに保存（ファイル保存しない）
- パスワードはbcryptでハッシュ化
- CORS設定で許可されたオリジンのみアクセス可
- 入力値はサーバー側でバリデーション

## テスト方針

Small / Medium / Big の3層でテストを実装する。

- **Small（単体テスト）**: モデルのバリデーション、ビジネスロジック
- **Medium（結合テスト）**: APIエンドポイント、認証フロー
- **Big（E2Eテスト）**: ブラウザでのユーザー操作テスト

自明な処理（単純なsetter等）以外はテストカバレッジを確保する。

## 設計方針

- 単一責任の原則を意識して、コントローラーは薄く保つ
- ビジネスロジックはモデルやサービスクラスに分離
- RESTfulなAPI設計
- Makefileでコマンドをまとめる
